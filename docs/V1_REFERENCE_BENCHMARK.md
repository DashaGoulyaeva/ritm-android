## Главный вывод

У «Ритма» сильная продуктовая ставка: он объединяет несколько ритмов в одном Today-экране. Но если смотреть как пользователь, который уже видел отдельные приложения, V1 должна закрыть не только "есть модуль", а зрелость каждого модуля:

- есть история;
- есть настройки;
- есть напоминания;
- есть понятные empty/fallback states;
- есть экспорт или контроль данных;
- есть релизная упаковка;
- нет stub-экранов в основных сценариях.

Если этого нет, сторонний человек сравнит не с нашим MVP-планом, а с приложениями, где одна фича уже доведена до нормального состояния.

## Сверка по доменам

### 1. Привычки vs Loop Habit Tracker

Что есть у референса:
- минималистичный быстрый интерфейс;
- гибкие расписания: не только daily, но и "3 раза в неделю", "через день";
- habit score / сила привычки;
- streak и история;
- детальная статистика и графики;
- отдельные напоминания на привычку;
- действие прямо из уведомления: отметить или отклонить;
- home screen widgets;
- экспорт CSV / SQLite;
- offline + privacy без аккаунта;
- нет рекламы и искусственных лимитов.

Что у нас уже хорошо:
- создание привычки;
- POSITIVE / AVOID тип;
- отметка за день;
- DONE / SKIPPED / FAILED;
- streak;
- история;
- локальное хранение;
- Today-интеграция.

Чего не хватает до V1-уровня:
- гибкое расписание привычки, хотя бы daily / selected weekdays / N раз в неделю;
- полноценные настройки привычек и архив без stub-ощущения;
- отдельное напоминание на привычку или понятный общий evening reminder;
- действие из уведомления: "выполнено" / "пропустить" желательно для V1, минимум для V1.1;
- статистика за период: неделя/месяц, процент выполнения;
- экспорт привычек в общем экспорте данных;
- widget можно оставить post-V1, но нельзя забыть: у Loop это сильный стандарт категории.

V1 baseline:
- пользователь может создать несколько привычек, задать расписание, получать напоминание, отмечать день, видеть streak/историю/процент, архивировать и не потерять данные.

### 2. Вода vs Quench

Что есть у референса:
- home screen для текущего прогресса;
- statistics screen;
- settings screen;
- notification screen;
- add notification screen;
- Room / Hilt / Navigation / WorkManager / notifications;
- публикация в Play Store.

Что у нас уже хорошо:
- быстрый лог воды/чая/кофе;
- цель в стаканах;
- история по дням;
- настройка цели;
- reminder через WorkManager;
- Today-интеграция.

Чего не хватает до V1-уровня:
- статистика воды хотя бы за 7/30 дней;
- нормальный экран управления напоминаниями, не только один выключатель;
- понятный режим "тихих часов" или временное окно напоминаний;
- отмена последней записи или удаление ошибочной записи;
- визуальный weekly summary для стор-скриншотов.

V1 baseline:
- пользователь может быстро отметить напиток, исправить ошибку, видеть прогресс сегодня, историю и простую статистику, настроить цель и напоминания.

### 3. Fasting vs Interfast

Что есть у референса:
- расчет eating/fasting period;
- сохранение выбранных настроек;
- time restricted feeding режимы: 16/8, 18/6, 20/4, 22/2;
- download/release через APK;
- заявленная privacy-позиция: free, open source, ads free, tracking free.

Что у нас уже хорошо:
- режимы 12/12, 14/10, 16/8, 18/6, 20/4, 23/1;
- ручной старт/стоп;
- активный таймер;
- история;
- уведомление о завершении;
- смена окна без удаления истории.

Чего не хватает до V1-уровня:
- настройка режима по умолчанию должна реально сохраняться;
- таймер должен быть устойчивым после закрытия/перезапуска приложения;
- понятная история: завершено / прервано / длительность / режим;
- подтверждение остановки сессии;
- мягкие тексты без медицинских обещаний;
- privacy/store-текст: fasting не является медицинской рекомендацией.

V1 baseline:
- пользователь запускает fasting, закрывает приложение, получает корректное уведомление, возвращается и видит правильный статус и историю.

### 4. Цикл vs EasyCycle2

Что есть у референса:
- Google Play публикация;
- Room + Compose;
- календарь;
- фазы;
- статистика;
- настройки;
- уведомления;
- подтверждающие диалоги и базовые shared UI-компоненты.

Что у нас уже хорошо:
- начало/конец периода;
- текущий день цикла;
- прогноз следующего периода;
- календарь;
- дневник дня;
- выделения, симптомы, настроение, заметка;
- осторожная продуктовая позиция без медицинских обещаний.

Чего не хватает до V1-уровня:
- настройки цикла: средняя длина цикла, длина периода, отображение/скрытие фертильных фаз;
- статистика по циклам: средняя длина, разброс, последние циклы;
- понятное редактирование прошлых дней;
- подтверждения для удаления/сброса данных;
- дисклеймер в UI / About.

V1 baseline:
- пользователь может вести календарь, редактировать прошлые записи, видеть прогноз и простую статистику, понимая, что это дневник, а не медицинский инструмент.

### 5. Цикл+ vs Petal

Что есть у референса:
- flow intensity;
- symptom levels: pain, cramps, cravings, headaches;
- mood level;
- cycle phase: menstrual, follicular, ovulation, luteal;
- prediction confidence;
- локальная база;
- признаки синхронизации / health-интеграций как более поздний уровень.

Что у нас уже хорошо:
- flow/выделения;
- симптомы;
- настроение;
- заметка;
- прогноз;
- календарь;
- локальное хранение.

Чего не хватает до V1-уровня:
- уровни симптомов, а не только список;
- confidence/уверенность прогноза или текст "нужно больше данных";
- явные фазы цикла в UI;
- больше структурированных симптомов: боль, спазмы, тяга/аппетит, головная боль;
- Health Connect для cycle/symptom-sync можно оставить post-V1; для шагов V1 использует Health Connect / Android Health APIs с fallback на системный сенсор.

V1 baseline:
- дневник цикла должен быть не просто заметкой, а структурированным логом состояния по дням: интенсивность, симптомы с уровнем, настроение, заметка, фаза, осторожный прогноз.

## Cross-domain требования, которые референсы задают для V1

Эти вещи повторяются у нескольких проектов и должны стать обязательными для «Ритма»:

| Требование | Почему обязательно |
|---|---|
| История по каждому ритму | Пользователь не ведет трекер ради одного дня; ценность появляется в динамике |
| Простая статистика | Без статистики приложение ощущается как форма ввода, а не помощник |
| Настройки без stub-экранов | В публичной версии каждый видимый switch обязан работать |
| Напоминания | Это стандарт категории для привычек, воды и fasting |
| Privacy-first объяснение | Loop и Interfast явно продают доверие; для «Ритма» это тоже сильная сторона |
| Экспорт/контроль данных | У Loop это зрелый стандарт; для «Ритма» минимум общий CSV/JSON export |
| Store/release упаковка | Quench, Loop, EasyCycle доступны в сторах; V1 должна иметь такой же уровень упаковки |
| Empty/fallback states | Особенно важно для шагов, цикла без истории, первых привычек и разрешений |

## Что добавить в V1_PUBLIC_LAUNCH_PLAN

### Must-have до V1

1. **Гибкие привычки.** Добавить расписание: daily / selected weekdays / N раз в неделю.
2. **Статистика привычек.** Минимум: процент выполнения за 7/30 дней + streak.
3. **Полноценный архив привычек.** Без stub-экрана.
4. **Водная статистика.** Минимум: 7/30 дней и среднее выполнение цели.
5. **Исправление ошибочной записи воды.** Удаление последней записи или удаление из истории.
6. **Управление уведомлениями.** Вода, привычки, fasting: включить/выключить, время, понятный статус.
7. **Fasting persistence.** Активная сессия корректно переживает закрытие приложения и смену дня.
8. **Цикл: настройки и статистика.** Длина цикла/периода, последние циклы, средняя длина.
9. **Цикл: уровни симптомов.** Минимум: нет/слабо/средне/сильно для ключевых симптомов.
10. **Prediction confidence.** Текстовая уверенность прогноза: "мало данных", "примерно", "на основе N циклов".
11. **Экспорт данных.** Минимум один общий экспорт CSV/JSON локальных данных.
12. **Privacy/About screen.** Версия, offline-first, no account, no ads/tracking, не медицинское приложение.
13. **Release/store контур.** Иконка, скриншоты, release signing, versioning, changelog, privacy policy.

### Should-have для V1, если хватает времени

1. Действия из уведомления привычки: выполнено / пропустить.
2. Home screen widget для привычек или Today summary.
3. Более красивые графики привычек/воды.
4. Dark mode.
5. Health Connect для шагов — V1 с fallback; для цикла и других health-sync сценариев — post-V1, если не ломает privacy-first простоту.

### Не блокировать V1

1. Облако и аккаунты.
2. AI-рекомендации.
3. Социальные функции.
4. Монетизация.
5. Wear OS.
6. Сложные медицинские сценарии.

## Обновленный V1 стандарт

V1 «Ритма» должна выигрывать у референсов не глубиной одной фичи, а цельностью: один спокойный Today-центр для нескольких ритмов. Но внутри каждого ритма нельзя быть слабее базового ожидания категории.

Минимальная планка:
- у привычек есть расписание, история, streak, статистика, архив, напоминания;
- у воды есть быстрый лог, история, статистика, цель, напоминания, исправление ошибок;
- у fasting есть режимы, таймер, persistence, история, уведомление;
- у цикла есть календарь, дневник, фазы, симптомы с уровнем, прогноз с уверенностью, статистика;
- у данных есть privacy-позиция и экспорт;
- у релиза есть нормальная store-упаковка.

Если пункт не закрыт, он должен быть явно помечен как V1.1 и не выглядеть в приложении как сломанная или недоделанная функция.
