# Module Decisions

Статус: черновик. Решения не утверждены.

Легенда:

- `KEEP` — можно оставить почти как есть.
- `REWORK` — смысл может пригодиться, реализацию пересмотреть.
- `DROP` — не переносить в MVP v2.
- `UNKNOWN` — решить после продуктовой рамки.

| Область | Решение | Комментарий |
|---|---|---|
| Gradle / multi-module shell | UNKNOWN | Оценить после выбора стратегии rebuild. |
| Hilt / DI | UNKNOWN | Вероятно сохранить, если остаёмся на Android/Kotlin. |
| Room / DataStore | UNKNOWN | Вероятно сохранить как offline-first базу. |
| Навигация | REWORK | Старый маршрутный граф не считать обязательным. |
| Today screen | UNKNOWN | Не переносить автоматически. Сначала определить новый главный сценарий. |
| Вода | UNKNOWN | Пересмотреть смысл и роль в новой концепции. |
| Привычки | UNKNOWN | Пересмотреть модель и место в продукте. |
| Цикл | UNKNOWN | Пересмотреть сценарий дневника и календаря. |
| Fasting | UNKNOWN | Пересмотреть необходимость в MVP v2. |
| Шаги | UNKNOWN | Не переносить автоматически. |
