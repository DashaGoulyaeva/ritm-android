# Product Reset v2

Статус: полный реворк и переосмысление.

Текущий MVP v1 сохранён как исследовательский прототип и технический архив. Он больше не является источником продуктовых требований для следующей версии.

- Baseline-тег: `mvp-v1-baseline`
- Рабочая ветка реворка: `codex/product-reset-v2`
- Код и документы v1 сохраняются, но считаются архивным материалом до отдельного решения.

## Правило реворка

Не исправляем старое приложение по инерции. Сначала заново формулируем смысл, сценарии и границы MVP v2, потом решаем, какие части кода оставить, переписать или убрать из активного продукта.

## Карта документов

- `docs/rework/UX_REVIEW_V1.md` — почему MVP v1 не попал в цель.
- `docs/rework/PRODUCT_V2.md` — новая продуктовая рамка.
- `docs/rework/MVP_SCOPE_V2.md` — новые границы MVP.
- `docs/rework/REBUILD_PLAN.md` — процесс полного реворка.
- `docs/rework/MODULE_DECISIONS.md` — что оставить, переписать или исключить.
