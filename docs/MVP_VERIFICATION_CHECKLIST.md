# MVP_VERIFICATION_CHECKLIST.md — Чек-лист верификации MVP

> Цель: проходить проверку одинаково на каждом APK и быстро фиксировать обратную связь.
> Формат: один запуск = один чек-лист + короткий отчёт.

---

## Подготовка устройства

1. Установить свежий APK (clean install для проверки first run).
2. Разрешить уведомления приложению.
3. Для сценариев шагов выдать `ACTIVITY_RECOGNITION`.
4. Проверить корректность даты, времени и таймзоны на устройстве.

---

## Обязательный smoke-чек

### HV-01 Первый запуск
> Модуль: готов. Onboarding screen реализован.
1. Открыть приложение после установки.
2. Убедиться, что стартовый экран загружается без падения.
3. Перезапустить приложение 2 раза.
4. Ожидаемо: нет crash, состояние не ломается.

### HV-02 Навигация
> Модуль: готов. Все 4 вкладки навигации реализованы.
1. Открыть основные разделы (Today, Привычки, Цикл, Настройки).
2. Вернуться на Today.
3. Ожидаемо: переходы работают, пустых/битых экранов нет.

### HV-03 Вода
> Модуль: готов. WaterQuickLogBottomSheet + WaterHistoryScreen.
1. На Today открыть быстрый лог воды.
2. Добавить воду, чай и кофе (по одному разу).
3. Открыть историю воды.
4. Ожидаемо: прогресс и история согласованы.

### HV-04 Напоминания воды
> Модуль: готов. WaterReminderWorker + WaterSettingsScreen.
1. В настройках воды включить reminder и поставить время +2 минуты.
2. Дождаться уведомления.
3. Выключить reminder.
4. Ожидаемо: уведомление приходит один раз, без дублей после выключения.

### HV-05 Привычки (после закрытия этапа 2)
> Модуль: готов. HabitsScreen + HabitDetailScreen + CheckHabitToday.
1. Создать привычку.
2. Отметить привычку выполненной за сегодня.
3. Открыть детали привычки.
4. Ожидаемо: статус/стрик/история согласованы.

### HV-06 Fasting (после закрытия этапа 3)
> Модуль: готов. FastingBottomSheet + FastingTimerWidget + FastingHistoryScreen.
1. Запустить сессию fasting (например, 16:8).
2. Проверить активный таймер и статус.
3. Остановить сессию.
4. Ожидаемо: запись в истории есть, состояние корректно завершено.

### HV-07 Цикл (после закрытия этапа 4)
> Модуль: готов. CycleCalendarScreen + CycleDayJournalScreen + PredictNextPeriod.
1. Отметить начало цикла.
2. Добавить дневниковую запись за день.
3. Открыть календарь цикла.
4. Ожидаемо: отметки и дневник видны в правильной дате.

### HV-08 Шаги (после закрытия этапа 4.5)
> Модуль: готов (с ограничением). StepsSensorDataSource требует ACTIVITY_RECOGNITION. Fallback: "Шагомер недоступен" если сенсор отсутствует.
1. Открыть Today и проверить блок шагов.
2. Походить 2-3 минуты и обновить экран.
3. Открыть историю шагов.
4. Ожидаемо: шаги растут, история по дням обновляется.

## Что фиксировать после каждого прохода

- Версия APK (имя файла + дата/время сборки).
- Устройство и версия Android.
- Результат каждого сценария HV: PASS/FAIL.
- Список багов с критичностью: blocker/high/medium/low.
- Итог: `Готово к следующему этапу` или `Нужна доработка`.
