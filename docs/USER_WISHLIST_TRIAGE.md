# USER_WISHLIST_TRIAGE.md — Разбор новых хотелок

> Цель: не потерять сильные идеи, но не раздуть V1 до комбайна. Каждая хотелка получает решение: V1, V1.1, V2 или Research.

---

## Принцип разбора

Хотелка попадает в V1, если она:
- усиливает главный экран Today;
- снижает когнитивную нагрузку;
- не требует аккаунтов/облака/сложных внешних контрактов;
- не превращает «Ритм» в медицинский, фитнес- или контентный продукт;
- может быть сделана законченно, без stub-ощущения.

Если идея классная, но требует интеграций, лицензий, контента, отдельного UX или меняет позиционирование, она уходит в V1.1/V2.

---

## Решения по хотелкам

| Хотелка | Решение | Что делаем | Что не делаем сейчас | Почему |
|---|---|---|---|---|
| Шаги: интеграция с Google Fit или похожим сервисом, отдельный экран | **V1 Must-have, но через Health Connect / Android Health APIs, не Google Fit** | Отдельный экран шагов, Today widget, история, источник данных, permission flow, fallback на системный сенсор | Не строим новую интеграцию на Google Fit API | Google Fit API deprecated и уходит к end-of-service в конце 2026; для Android step tracking правильнее Health Connect |
| Курение/алкоголь как отдельные экраны или выделенные привычки | **V1.1 Product slice; в V1 — подготовить через preset habits** | В V1 добавить модель/UX для preset habits: цвет, фон, иконка, подсказка, кастомный reminder text | Не делать отдельные экраны курения/алкоголя в V1 | Это сильная идея, но отдельные экраны требуют тонкого UX и осторожного текста без медицинских обещаний/стыда |
| Предустановленные привычки с красивыми фонами, подсказками, флоу | **V1 Should-have** | Templates для привычек: сон, вода, прогулка, курение, алкоголь, медитация; визуальные акценты и подсказки | Не делать большой marketplace привычек | Это усиливает onboarding и помогает начать без пустого экрана |
| Кастомные напоминания с любым текстом | **V1 Must-have** | Пользователь может задать текст reminder для привычки/ритма; есть мягкий default text | Не делать сложные сценарии/условия/автоматизации | Это маленькая фича с высокой ценностью: напоминание становится личным |
| Интеграция с музыкальными сервисами: “в каком ты сегодня ритме” | **V2 Research / Differentiator** | Исследовать Spotify/Яндекс Музыку/YouTube Music: можно ли открывать playlist/deeplink по mood/steps | Не делать полноценную интеграцию в V1 | Музыкальные API, аккаунты, права, UX и зависимости могут съесть фокус V1 |
| Музыка под шаги или настроение | **V1.1 Lite concept, если без интеграций** | Можно позже сделать “ритм дня” + ручная рекомендация/playlist link без аккаунта | Не синхронизировать музыку/темп/шаги в V1 | Идея красивая для бренда, но не базовая надежность |
| Медитация как отдельный экран | **V1.1 / V2** | После V1 можно добавить простой “тихий экран”: таймер дыхания/медитации, история минут, напоминание | Не добавлять полноценную библиотеку медитаций в V1 | Отдельный экран меняет scope. В V1 можно оставить медитацию как preset habit |
| Интеграция с часами: хотя бы показывать уведомления | **V1 Must-have как wearable-friendly notifications** | Делать обычные Android notifications так, чтобы они нормально отображались на часах: короткий заголовок, понятный текст, grouped channels, actions где уместно | Не делать отдельное Wear OS-приложение в V1 | Это даёт ощущение взрослой интеграции без отдельного watch-продукта и без раздувания scope |

## Детализация по шагам

### Что делаем в V1

- отдельный экран «Шаги»;
- Today widget;
- история по дням;
- источник данных: Health Connect, если доступен;
- fallback: системный `TYPE_STEP_COUNTER`;
- fallback UI: нет разрешения, нет сенсора, Health Connect недоступен;
- объяснение в onboarding/settings;
- без спортивных целей, калорий и соревнований.

### Почему не Google Fit

Google Fit API deprecated; официальная рекомендация для mobile-first step tracking — Health Connect / Android Health APIs. Поэтому в пользовательском языке можно писать “интеграция со здоровьем/шагами на телефоне”, но в техническом плане не строить V1 на Google Fit.

## Детализация по курению и алкоголю

### Продуктовая идея

Курение и алкоголь не должны быть просто обычными привычками в сером списке. Это чувствительные сценарии, где важны:
- отсутствие стыда;
- видимость прогресса;
- мягкие подсказки;
- возможность сорваться и вернуться;
- отдельный визуальный язык.

### V1 решение

В V1 делаем основу:
- `HabitTemplate` / preset habits;
- специальные visual presets: фон, цвет, иконка, подсказка;
- default reminder copy;
- возможность изменить reminder text;
- тип привычки `AVOID` работает для “не курить” / “без алкоголя”.

### V1.1 решение

После стабильной V1 можно делать отдельные экраны:
- “Курение” — счётчик дней без курения, сорвался/вернулся, заметки о триггерах;
- “Алкоголь” — dry days, мягкая статистика, напоминания перед рискованными днями;
- без медицинских обещаний и без морализаторства.

## Детализация по кастомным напоминаниям

### V1 Must-have

Каждое напоминание должно иметь:
- включить/выключить;
- время;
- дефолтный текст;
- пользовательский текст;
- preview уведомления;
- возможность вернуть дефолт.

Ограничение V1:
- без сложных правил “если/то”;
- без геолокации;
- без синхронизации с календарём;
- без повторов по сложным cron-паттернам.

## Детализация по музыке

### Почему идея сильная

“В каком ты сегодня ритме?” — это очень брендовая фича. Она может отличать «Ритм» от сухих трекеров.

### Почему не V1

Музыкальная интеграция тянет:
- OAuth/аккаунты;
- разные сервисы;
- права и ограничения API;
- нестабильные deeplink-и;
- поддержку состояний “сервис не установлен”;
- риск превратить приложение в развлекательный продукт.

### Возможный V1.1/V2 путь

- сначала без интеграций: “ритм дня” как настроение/темп;
- потом playlist deeplink;
- потом интеграция с одним сервисом;
- только если это не ломает privacy-first позицию.

---

## Детализация по медитации

### V1 решение

Медитация входит как preset habit:
- “помедитировать 5 минут”;
- reminder text;
- отметка выполнения;
- streak.

### V1.1/V2 решение

Отдельный экран возможен позже:
- простой таймер;
- дыхательный ритм;
- история минут;
- мягкие напоминания;
- без библиотеки аудио на старте.

## Итог

Обновлённая логика:

1. **Шаги с Health Connect** — да, V1, отдельный экран.
2. **Кастомные напоминания** — да, V1.
3. **Preset habits** — желательно V1, потому что помогает onboarding и будущим smoking/alcohol флоу.
4. **Курение/алкоголь как отдельные экраны** — V1.1, сначала через preset habits.
5. **Музыка** — V2 Research, брендовая, но не базовая.
6. **Медитация отдельным экраном** — V1.1/V2; в V1 как preset habit.
7. **Часы** — V1 на уровне корректных wearable-friendly уведомлений; отдельное Wear OS приложение позже.
