# MINI-SPEC F-07 — Quality Gates Baseline

**Статус:** Принято  
**Дата:** 2026-04-20  
**Роль:** Product Agent

## User story

Как команда разработки, мы хотим единый quality-gate pipeline для проекта,
чтобы каждый следующий vertical slice проверялся одинаково и не ломал MVP
до интеграции в общий экран Today.

## Acceptance criteria

1. В проекте доступны проверки `lint`, `ktlintCheck`, `detekt`, `testDebugUnitTest`.
2. Есть единая команда уровня root, запускающая весь набор проверок.
3. Проверки не меняют продуктовое поведение и не добавляют MVP-фичи.
4. Правила качества совместимы с текущим Kotlin/Compose/Hilt стеком.
5. Definition of Done для нового slice включает:
   - код компилируется,
   - релевантные unit-тесты проходят,
   - quality checks проходят без критических ошибок,
   - `TASKS.md` и `docs/devlog.md` синхронизированы.

## Edge cases

- Если часть окружения недоступна (например, Android SDK), задача получает статус `BLOCKED` с явной причиной.
- Если линтер/статический анализ выявляет ошибки в старом коде, исправления выполняются только в рамках slice или отдельным infra-slice.
- Если плагинная интеграция (GitHub/Figma) недоступна, работа продолжается локально с фиксацией ограничений.
