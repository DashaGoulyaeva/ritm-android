# ADR-004: Контракты для воды/настроек/reminder (W-06, W-08, S-02)

**Статус:** Принято  
**Дата:** 2026-04-20

## Контекст

Нужно добавить историю воды, локальные reminder-уведомления и управление ими из настроек,
не нарушая правило архитектуры: feature-модули не зависят друг от друга напрямую.

## Решение

1. `feature-water` отвечает за:
   - историю воды (`WaterHistoryScreen`),
   - Worker уведомления (`WaterReminderWorker`).
2. `feature-settings` отвечает за:
   - UI настроек воды (цель, включение reminder, время),
   - работу с DataStore-ключами воды/reminder.
3. Связь настроек и планировщика reminder идёт через интерфейс в `core-common`:
   - `WaterReminderScheduler` (контракт),
   - реализация в `app` (DI), чтобы не нарушать границы feature-feature.
4. Публичные контракты `feature-water` не ломаем:
   - `WaterRepository`, `WaterGoalRepository`, `WaterUiState`, существующие use cases остаются совместимыми.

## Последствия

- (+) Архитектурные границы модулей сохраняются.
- (+) Reminder-планирование централизовано и не размазано по UI-слоям.
- (+) Settings может управлять reminder без прямой зависимости от `feature-water`.
- (-) Для полноценной верификации reminder нужен SDK/девайс и разрешения уведомлений.
