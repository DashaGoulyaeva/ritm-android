# ADR-005: Контракты для модуля шагов (Steps, Этап 4.5)

**Статус:** Принято (ретроспективно)
**Дата:** 2026-04-25

## Контекст

Нужно реализовать подсчёт шагов за текущий день и хранение истории по дням без подключения к Google Fit или Health Connect. Требования MVP: работать offline, использовать только встроенные API Android, не добавлять внешних зависимостей. Реализация живёт в `feature-home` — отдельный модуль `feature-steps` не создаётся, так как Steps является одним из виджетов Today screen и не требует отдельного точки входа в навигации верхнего уровня.

## Решение

1. **Источник данных** — `SensorManager.TYPE_STEP_COUNTER`: кумулятивный счётчик с момента загрузки устройства. Ежедневный базовый сброс рассчитывается в `StepsRepository` путём сохранения значения счётчика на начало дня.

2. **Разрешение** — `ACTIVITY_RECOGNITION` (обязательно для Android 10+). При `minSdk = 26` разрешение запрашивается всегда на старте. Если разрешение не выдано — возвращается fallback.

3. **Fallback** — если датчик физически недоступен или разрешение не выдано, репозиторий эмитит `-1`. ViewModel интерпретирует `-1` как состояние `SensorUnavailable`; виджет показывает сообщение «Шагомер недоступен».

4. **Персистентность** — каждая эмиссия датчика автоматически сохраняется в `StepDailyEntity` (Room). Хранится одна запись на день; при обновлении запись перезаписывается. Это обеспечивает историю шагов по дням без дополнительного фонового сервиса.

5. **Размещение кода** — весь Steps-код (`StepDailyEntity`, `StepsDao`, `StepsSensorDataSource`, `StepsRepository`, use cases, `StepsViewModel`, виджет, экран истории) живёт в `feature-home`. Нет отдельного `feature-steps` модуля.

## Последствия

- (+) Нет зависимости от Google Play Services или Health Connect — приложение работает на любом Android 8+ устройстве.
- (+) Архитектура остаётся простой: Steps не выходит за пределы `feature-home`.
- (+) Fallback покрывает оба сценария (нет датчика, нет разрешения) единым механизмом.
- (-) Кумулятивный счётчик сбрасывается при перезагрузке устройства; если перезагрузка произошла в течение дня, шаги за день будут неполными. Принято как приемлемое ограничение MVP.
- (-) Нет синхронизации с Health Connect — шаги из других приложений не учитываются (post-MVP).
