# DESIGN-AUDIT-002 — Направление для дизайн-версии v3

## Статус: Draft | Дата: 2026-04-21

---

## 0. Зачем этот документ

Этот документ фиксирует решения для **перерисовки v3** на основе:

- аудита v2;
- пользовательских наблюдений (6 пунктов);
- продуктовых ограничений MVP.

Цель: чтобы по этому файлу можно было сразу отрисовывать `design_v3` без двусмысленностей.

---

## 1. Входные наблюдения (обработано)

### 1) «Шрифты нравятся только в баннерах, в остальном слишком жирно»

**Решение для v3**
- Оставить выразительный `Space Grotesk` только для hero-баннеров и крупных чисел.
- Для всего остального UI перейти на более спокойную шкалу Inter:
  - секционные заголовки: `600` (вместо 700/800),
  - карточки/лейблы: `500`,
  - вторичный текст: `400`.
- Снизить количество ALL CAPS и плотных трекингов в мелком тексте.

### 2) «Жирные шрифты и тонкие линии не связаны»

**Решение для v3**
- Свести типографику и линии в единую «массу штриха»:
  - если текст `600+`, линия не тоньше `1.5`;
  - если линия `1.0–1.2`, текст рядом не тяжелее `500`.
- Базовые толщины:
  - контур surface/card: `1.25`,
  - активный контур/фокус: `1.75`,
  - вспомогательные разделители: `1.0` (пунктир/soft).
- Добавить один мягкий курсивный стиль только для аннотаций/подсказок, не для основного UI.

### 3) «Волнистые подчёркивания нравятся, но не везде»

**Решение для v3**
- Убрать постоянные волнистые подчёркивания из всех заголовков.
- Ввести состояние **активного контекста**:
  - волна появляется только у заголовка активного блока/экрана;
  - появляется при фокусе, тапе, открытии sheet/dialog;
  - исчезает на неактивных секциях.
- Волна становится сигналом «ты сейчас здесь», а не декоративным шумом.

### 4) «Дизайн слабо ассоциируется с ритмом»

**Решение для v3**
- Ввести явную концепцию: **«Ритм = пульс дня из 5 дорожек»**.
- На Today сделать главный паттерн ритма не текстом, а формой/поведением:
  - микро-пульс, такт, повторяемость, синхронность индикаторов.
- В интерфейсе закрепить лексику ритма:
  - «такт дня», «сбился ритм», «ровный ритм», «пульс недели».

### 5) «Цветовое разделение нравится, но не все цвета понятны (шаги жёлтые?)»

**Решение для v3**
- Зафиксировать семантику цветов в отдельном блоке прямо в макете:
  - Цикл — `#D81B60` (биология/фаза),
  - Вода — `#29B6F6` (вода/чистота),
  - Fasting — `#FF8A65` (метаболическое окно),
  - Привычки — `#66BB6A` (рост/закрепление),
  - Шаги — `#FDD835` (энергия/движение/дневной свет).
- Для шагов оставить жёлтый, но снизить «кислотность» в UI-слоях:
  - жёлтый как акцент, а не как большой фон с белым текстом.

### 6) «Кольца очень прикольные, хочу сохранить + идея музыки post-MVP»

**Решение для v3**
- Не выбрасывать кольца, а встроить в лучший сценарий Today (V1):
  - сделать **гибрид**: верхний блок V1 + компактный ритм-модуль с кольцами.
- Кольца в MVP статичные/состояния, в post-MVP — анимируемый «ритм-плеер».
- Заложить в v3 визуальный фундамент для будущей музыкальной синхронизации (без реализации в MVP).

## 2. Главная концепция v3

### «Ритм дня: стек действий + пульс состояния»

- **Стек** даёт понятность и скорость (MVP-ядро).
- **Пульс (кольца)** даёт фирменность и связь с названием.

Итог: не выбираем «или красиво, или понятно» — соединяем оба слоя.

## 3. Ключевые решения v3 (обязательные)

### CRITICAL

1. Пересобрать типографическую систему: hero ≠ весь UI.
2. Убрать белый текст на светлых акцентных фонах (особенно steps/water/fasting).
3. Ввести правила активного подчеркивания (только по фокусу/контексту).

### HIGH

4. Today сделать гибридным: `Stack + Rhythm Rings`.
5. Закрепить легенду семантики цветов в одном месте (в макете и handoff).
6. Снизить визуальный шум (толщины, капсы, перегруз контуров).

### MEDIUM

7. Привести микротексты к «ритм-лексике».
8. Добавить в v3 аннотации, что подготовлено под post-MVP музыку (без фиче-крипа).

## 4. Спецификация для отрисовки v3

### 4.1 Typography

- `Space Grotesk`:
  - только hero-заголовки баннеров;
  - крупные KPI-числа (время/шаги/проценты).
- `Inter`:
  - все секции, карточки, навигация, формы.
- Весовые ограничения:
  - Display: 700–800,
  - Section title: 600,
  - Body primary: 500,
  - Body secondary: 400.

### 4.2 Линии и контуры

- Базовый контур: `1.25`.
- Active/focus: `1.75`.
- Hairline/разделители: `1.0`.
- Запрет: сочетание `font-weight: 700+` с `1.0` линиями в одном компоненте.

### 4.3 Волнистые подчёркивания

- По умолчанию: выключены.
- Включаются только:
  - у активной секции после тапа,
  - в открытом bottom sheet,
  - в выделенном фокусе внутри сценария.
- Длительность индикации: краткоживущая (как маркер контекста).

### 4.4 Цвет и контраст

- Сохранить модульную палитру.
- Для светлых акцентов использовать тёмный текст `ink`, не белый.
- Жёлтый (шаги) — как энергия движения:
  - можно использовать более спокойный фон варианта `steps-soft` для больших поверхностей,
  - насыщенный жёлтый оставить для индикаторов прогресса/маркеров.

## 5. Today v3: гибридная схема (основа макета)

### Блок A — Hero summary (как в V1)
- Один понятный баннер со статусом дня.
- Ниже — 5 карточек ритмов (tap-first).

### Блок B — Rhythm rings (из V3, но компактно)
- Небольшой модуль «пульс дня» между баннером и карточками или в сворачиваемом хедере.
- Кольца отображают 5 ритмов цветом и прогрессом.
- Тап по кольцу = переход к соответствующему модулю.

### Почему это работает
- Пользователь получает привычный MVP-поток (карточки/быстрые действия).
- Бренд «ритма» виден сразу, без перегруза и без потери понятности.

## 6. Post-MVP мост к музыке (заложить сейчас, не реализовывать сейчас)

### Что заложить в v3
- Визуальный язык «пульса»:
  - фазные кольца,
  - повторяемые такты,
  - мягкие пульсации состояний.
- Нейминг на экране шагов:
  - «Шагай в своём ритме» как будущий hook для sync.

### Что НЕ делать в MVP
- Реальная синхронизация с музыкальным плеером.
- Эквалайзер в реальном времени.
- Любые внешние интеграции.

---

## 7. Что должно получиться на выходе design_v3

1. Один финальный Today (гибрид), без трёх конкурирующих концепций.
2. Типографика спокойнее, баннеры остаются выразительными.
3. Подчёркивания работают как интерактивный сигнал, не как фон.
4. Цвета понятны семантически и читаются по контрасту.
5. Кольца сохранены как фирменный слой и фундамент под post-MVP музыку.

---

## 8. Definition of Done для v3 (дизайн-итерации)

- Есть обновлённая типографическая шкала с весами и правилами применения.
- Есть Today-гибрид в одном варианте.
- Есть состояние активного подчёркивания на примере минимум 3 сценариев.
- Есть явная легенда семантики 5 цветов.
- Нет экранов, где светлый фон + белый текст даёт плохую читаемость.
- Отдельно помечены MVP vs post-MVP элементы музыкальной концепции.

---

## 9. Рекомендованный порядок перерисовки

1. Токены типографики и линий.
2. Today v3 (гибрид + активные состояния).
3. Шаги (с обновлённой цветовой логикой и ритм-лексикой).
4. Остальные модули по новой типографике.
5. Финальная проверка контраста/читабельности/консистентности.
