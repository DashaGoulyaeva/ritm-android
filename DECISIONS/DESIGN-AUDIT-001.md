## 2. Плюсы

- **Охват MVP почти полный.** За двумя исключениями все ключевые экраны из MVP_SCOPE.md отражены — нет «белых пятен» в основных потоках.
- **Четыре варианта Today.** Наличие альтернатив (стек / кольца / лента / фраза) позволяет принять осознанное решение до старта реализации. Это хорошая практика design exploration.
- **TodayEmpty / first-run state.** Отдельно прорисован пустой state — критичен для первого запуска; большинство ранних вайерфреймов его пропускают.
- **Цикл покрыт полностью.** CycleCalendar + CycleDayJournal покрывают весь флоу (календарь → дневник за 30 сек). Прогноз сформулирован осторожно («~4 мая», без «точного дня») — соответствует принципу «без ML-обещаний».
- **Привычки: POSITIVE/AVOID в одном списке.** Флоу чёткий — хекмап 6 недель + стрик + настройки в одном экране HabitDetail. Быстрый лог (тап = DONE, long-press = SKIPPED/FAILED) аннотирован.
- **Fasting: информативные этапы без медицинских обещаний.** Note «без медицинских обещаний · информационно» прямо в макете — соответствует PRODUCT.md.
- **Вода: только стаканы, никаких мл.** Соответствует ADR-002. Три типа (вода/чай/кофе) задокументированы.
- **Онбординг помечен как post-MVP.** Автор сам зафиксировал это в Note и в HTML — дисциплина.
- **Навигация: 4 таба** (Сегодня / Цикл / Привычки / Ещё). Fasting и Вода доступны через Today — соответствует описанному в задании.
- **Аннотации-заметки в дизайне.** Компонент `Note` использован для объяснения UX-намерений прямо в макете — снижает вопросы при хендовере.

## 3. Проблемы и минусы

### CRITICAL

- **[CRITICAL] Цветовая схема полностью не соответствует проектной.** Весь драфт выполнен в ч/б (INK = `#1a1a1a`, PAPER = `#fbf8f3`) с двумя акцентами — `oklch(68% 0.14 35)` (тёплый коралловый, ~`#cc7b5a`) и `oklch(72% 0.09 220)` (холодный голубой). Проектная палитра: Primary `#5C6BC0` (индиго), WaterAccent `#29B6F6`, CycleAccent `#D81B60`, FastingAccent `#FF8A65`, HabitsAccent `#66BB6A`. Акценты не совпадают ни по оттенку, ни по насыщенности. При переводе в Compose это потребует полного переосмысления цветовой системы.
- **[CRITICAL] Шрифты wireframe-специфические (Caveat, Architects Daughter) — в реализации недопустимы.** Material3 на Android использует Roboto/нативные шрифты. Итоговый дизайн должен быть переведён на M3 Typography scale (displayLarge, titleMedium, bodySmall и т.д.). Сейчас это не указано ни в одном файле.

### HIGH

- **[HIGH] FastingBottomSheet — отрисован как full-screen, а не bottom sheet.** По MVP-контексту и навигационной схеме («Fasting доступен через Today bottom sheet») выбор режима должен открываться поверх Today, а не заменять экран. В FastingPicker нет ни gesture bar, ни drag-handle, ни overlay — нет признаков bottom sheet-паттерна.
- **[HIGH] WaterHistoryScreen — полностью отсутствует.** Экран «История воды по дням» входит в MVP_SCOPE.md, но в драфте его нет. Единственное — мелкий лог за сегодня в WaterSheet и ссылка «посмотреть всё →» без назначения.
- **[HIGH] WaterSettingsScreen — отсутствует отдельным экраном.** В SettingsScreen есть пункт «цель по воде», но экран настройки воды (цель в стаканах + напоминание) MVP-ом выделен отдельно. Не прорисован.
- **[HIGH] FastingHistoryScreen — только заглушка.** Три строки истории встроены в FastingPicker. Отдельного экрана нет — при росте данных такой подход не масштабируется, и экран в MVP заявлен отдельно.
- **[HIGH] Нет описания переходов и навигации.** Wireframe показывает статичные состояния, но не объясняет: как TodayV2 (кольца) открывает WaterSheet? Какой gesture? Как происходит back-навигация с CycleDayJournal (кнопка «× закрыть» или swipe-down)? Это должно быть аннотировано.

### MEDIUM

- **[MEDIUM] TodayV2 (кольца) — концепция нечитаемая без легенды.** Четыре кольца без лейблов прямо на кольцах — пользователь не поймёт, что есть что, без взгляда на легенду снизу. На телефоне кольца будут ещё меньше (в коде: viewBox 220×220 внутри PhoneFrame 320×640). Пояснение Note «4 кольца = 4 ритма» не решает UX-проблему.
- **[MEDIUM] TodayV3 (лента времени) — сложно для первого взгляда.** Вертикальная временная линия — нестандартный паттерн для «статус дня». Пользователю нужно прокручивать, чтобы увидеть будущие события. Нарушает принцип «данные на виду».
- **[MEDIUM] HabitDetail: легенда heatmap на английском** — «done», «skipped», «failed», «empty» при том, что весь остальной UI на русском. Нарушение языковой консистентности.
- **[MEDIUM] CycleDayJournal: «× закрыть» — неоднозначный паттерн.** Экран показан как full-screen, но по продуктовой спецификации это bottom sheet. Нет drag-handle, нет scrim-фона.
- **[MEDIUM] WaterSheet: разнородные цвета для типов напитков.** Чай — `oklch(75% 0.06 50)`, кофе — `oklch(62% 0.04 30)`. В проектной палитре этих цветов нет. Нужно либо определить их или использовать нейтральные оттенки.
- **[MEDIUM] Fasting: нет экрана/кнопки «новая сессия» после завершённой.** FastingActive показывает таймер с кнопками «пауза» и «стоп», но что происходит после стопа — не показано. Переход к FastingPicker или к истории?
- **[MEDIUM] BottomNav не показывает иконки — только квадраты/кружки-заглушки.** Для хендовера в Compose нужно решить: какие именно иконки (Material Symbols или кастомные). Сейчас это не обозначено.

### LOW

- **[LOW] PhoneFrame (320×640) меньше реальных устройств.** AndroidDevice задан 412×892 (корректные M3 размеры), но PhoneFrame в wireframes — 320×640. При переводе в реальный UI возможно переполнение контента.
- **[LOW] MD_C в android-frame.jsx — Material 3 цвета teal/green (`#006a60`)** — не связаны с проектной схемой индиго. Этот файл, судя по всему, шаблонный и не адаптирован к проекту.
- **[LOW] Нет состояния ошибок и edge cases.** Что если у пользователя 0 циклов? Что если fasting начат, а данных нет? Только TodayEmpty как first-run state — остальные пустые состояния не показаны.
- **[LOW] Onboarding показывает только шаг 2/4.** Если решат включить в MVP — нужны все 4 шага и логика пропуска.
- **[LOW] SketchCheck использует символы ✓, –, × без кастомных иконок** — в Compose нужны конкретные Icon (@DrawableRes или Material Icons).

## 5. Рекомендации по экранам

### Today (все 4 варианта)

- **V1 (стек)** — самый безопасный и понятный. Рекомендуется как baseline MVP. Четкий порядок: цикл → fasting → вода → привычки. Доработать: добавить tappable область для открытия bottom sheet на каждой карточке.
- **V2 (кольца)** — концептуально сильный, но требует онбординга. Для MVP без онбординга — рискованно. Если оставлять — добавить лейблы прямо на дугах или заменить на Activity Rings (Apple-like) с цветами каждого ритма.
- **V3 (лента)** — нестандартно, высокий когнитивный барьер при первом открытии. Лучше как опциональный вид, не дефолтный.
- **V4 (фраза)** — оригинально, работает как «резюме дня» после заполнения. Слабо для empty state. Можно использовать как виджет в V1, не как полный экран.
- **Рекомендация:** взять V1 как основу MVP с элементами V4 (статусная фраза над карточками).

### CycleCalendar

- Добавить: возможность тапнуть по дню в прошлом для редактирования. Сейчас только «открыть день →» для текущего.
- Добавить: кнопку «+ отметить начало» для логирования первого дня менструации.
- Прогноз «~4 мая» — хорошо. Добавить disclaimer «на основе N циклов».

- Настроение: emoji-символы (`꒰`, `‿`, `◡`) — нестандартные символы, могут рендериться по-разному на устройствах. Заменить на Material Icons или кастомные иконки.

### HabitsList

- Хорошо. Добавить: swipe-to-mark (swipe вправо = DONE) как альтернативу тапу — стандартный паттерн.
- Фильтры «сегодня / все / архив» — хорошо, соответствует MVP_SCOPE.

### HabitDetail

- Легенду heatmap перевести на русский: «выполнено / пропущено / не выполнено / нет данных».
- Настройки в HabitDetail (частота, напоминание, тип, архивировать, удалить) — правильное место.

### HabitCreate

- Добавить: валидацию — нельзя создать без названия.

- Добавить визуальное разграничение «активная сессия голодания» — если fasting идёт, кофе/чай попадают в окно и это нужно отметить.

## 6. Цветовая схема

### Текущая схема в драфте

| Переменная | Значение | Назначение в драфте |
|---|---|---|
| INK | `#1a1a1a` | Основной текст, рамки |
| PAPER | `#fbf8f3` | Фон |
| PAPER2 | `#f2ede3` | Альтернативный фон |
| ACCENT | `oklch(68% 0.14 35)` ≈ `#cc7b5a` | Корал — цикл + primary accent |
| ACCENT2 | `oklch(72% 0.09 220)` ≈ `#7ab8d4` | Голубой — вода |
| HIGHLIGHT | `oklch(92% 0.08 95)` ≈ `#f5f0d0` | Бледно-жёлтый highlight |

### Проектная схема

| Роль | Значение | Назначение |
|---|---|---|
| Primary | `#5C6BC0` | Индиго — основной цвет приложения |
| WaterAccent | `#29B6F6` | Голубой — вода |
| CycleAccent | `#D81B60` | Розово-красный — цикл |
| FastingAccent | `#FF8A65` | Персиковый — голодание |
| HabitsAccent | `#66BB6A` | Зелёный — привычки |
| Background | `#F5F5F5` | Нейтральный серо-белый |
| Surface | `#FAFAFA` | Поверхность карточек |

### Расхождения

1. **ACCENT ≈ `#cc7b5a`** (тёплый коралл) используется и для цикла, и как primary. В проектной схеме цикл — `#D81B60` (насыщенный розово-красный), а primary — `#5C6BC0` (индиго). Это принципиальное расхождение.
2. **ACCENT2 ≈ `#7ab8d4`** (приглушённый голубой) vs WaterAccent `#29B6F6` (яркий голубой). Оттенки близкие, но яркость разная — при финальной реализации нужно унифицировать.
3. **Fasting в драфте** — использует INK (чёрный) для дуги таймера. Проектная схема выделяет fasting персиковым `#FF8A65` — это важный сигнал пользователю.
4. **Привычки в драфте** — используют ACCENT (коралловый) для стрика и check. Проектная схема — зелёный `#66BB6A`.
5. **Background/Surface** — PAPER `#fbf8f3` (тёплый кремовый) vs проектный `#F5F5F5` (холодный нейтральный). Визуально разные атмосферы — «дневник» vs «чистый UI».

### Итог по цвету

Wireframe использует минималистичную ч/б схему с одним акцентом — это оправдано для lo-fi wireframes, НО при переводе в Compose Color Scheme необходимо:

1. Принять Color Scheme проекта как единственный источник истины.
2. Назначить акцентный цвет каждому ритму согласно проекту (цикл = `#D81B60`, вода = `#29B6F6`, fasting = `#FF8A65`, привычки = `#66BB6A`).
3. Primary `#5C6BC0` использовать для BottomNav active state, FAB, кнопок подтверждения.
4. Background `#F5F5F5` + Surface `#FAFAFA` как нейтральная основа.
5. Тёплый PAPER (`#fbf8f3`) можно сохранить как альтернативный surface variant для «мягкости».

## 7. Plan доработки

### Приоритет 1 — Перед началом реализации (блокирующие)

1. **Принять решение по варианту Today.** Рекомендуется V1 (стек) как MVP baseline. Зафиксировать в DECISIONS/ADR.
2. **Создать Color Scheme документ** с маппингом: `ACCENT` → `CycleAccent`, `ACCENT2` → `WaterAccent`, добавить FastingAccent и HabitsAccent. Привязать к M3 ColorScheme (primary, secondary, tertiary).
3. **Зафиксировать Typography Scale** — какой шрифт, какие размеры. Перевести Caveat/Architects Daughter → конкретные M3 TextStyle (headlineMedium, bodyLarge и т.д.).

### Приоритет 2 — Дорисовать недостающие экраны

4. **WaterHistoryScreen** — список по дням + недельный summary.
5. **WaterSettingsScreen** — цель + reminder.
6. **FastingHistoryScreen** — отдельный список сессий.
7. **FastingPicker как bottom sheet** — переоформить с drag-handle и overlay scrim.
8. **CycleDayJournal как bottom sheet** — то же.

### Приоритет 3 — Доработать существующие экраны

9. **Today V1** — добавить tappable affordance на каждой карточке (иконка «›» или ripple зона).
10. **HabitDetail** — перевести легенду heatmap на русский.
11. **FastingActive** — добавить цвет FastingAccent на прогресс-дугу.
12. **SettingsScreen** — добавить раздел Цикл, доработать «стереть всё» → AlertDialog паттерн.
13. **BottomNav** — выбрать конкретные иконки (Material Symbols Outlined рекомендуется).

### Приоритет 4 — Аннотации и переходы

14. **Добавить flow-диаграммы** — как Today открывает WaterSheet/FastingPicker (тап по карточке? FAB?).
15. **Описать back-navigation** для bottom sheets (swipe-down vs кнопка).
16. **Документировать empty states** для всех экранов, не только Today.

### Приоритет 5 — Post-MVP (не блокирует)

17. Onboarding: дорисовать шаги 1, 3, 4.
18. Тёмная тема (color roles для night mode).
19. Edge cases: 0 привычек, 0 циклов в истории, первый fasting.
